# Lockfile

`addons.lock` is generated by `gpm` and should be committed with
`addons.toml`. It records what was actually resolved and installed.

Each entry contains:

| Field | Meaning |
| --- | --- |
| `resolved_version` | Git commit SHA, GitHub release tag, or archive version marker. |
| `source_path` | Source subdirectory that was installed. |
| `checksum` | SHA-256 for archive and GitHub release downloads. |
| `spec_hash` | Hash of the manifest fields that affect resolution. |

## Install Behavior

`gpm install` compares the current manifest entry with the lock entry:

- If the hash matches, the lock pin is honored.
- If the hash differs or the entry is missing, the addon is re-resolved.
- On a full install, lock entries for addons no longer present in the manifest
  are removed.

For Git sources, a locked install fetches the exact commit SHA stored in
`resolved_version`.

For archive and GitHub release sources, checksums are verified when they are
available. A manifest `checksum` is checked on every fetch. A lockfile
`checksum` is checked when installing from an existing lock pin.

## Update Behavior

`gpm update` ignores existing pins for the selected addons:

```bash
gpm update
gpm update dialogue_manager
```

Use `gpm update` when you intentionally want to move to newer refs or refresh
release/archive pins. Review the resulting `addons.lock` diff before merging.

## Remove Behavior

`gpm remove <name>` deletes the installed addon directory, removes the manifest
entry, and removes any lock entry for that addon.
